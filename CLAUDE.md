# つづく（tsuduku）

仲間と続ける運動ログ共有アプリのプロトタイプ。次セッション以降もこの方針を維持すること。

## コンセプト
- **仲間と一緒に運動を「続ける」** ことが核。SNS的な運動ログの共有アプリ。
- **予定の宣言が核心機能**：「きょう誰が何をするか」を先に宣言する＝サボれない仕組み。
- 3画面構成：
  - **記録（Feed）**：仲間の運動投稿。部位タグ・実施データ（実施時間/消費/摂取/メンテ差分）・リアクション（🔥💪👏）。
  - **予定（Schedule）**：週ストリップ、今週の予定達成率リング、きょうの宣言、予定リスト（実施済/予定変更/未実施）。
  - **のび（Progress）**：本人ファースト。今週の運動日数/連続記録/体重＋サブ指標（総運動時間/予定達成率/メンテ差分）、体重折れ線、運動日数の棒グラフ、最下部に「仲間のうごき（今週）」。

## デザイン方針
- **あすけん / hacomono 系のミニマル**。白基調・余白広め・ヘアライン主体。
- **アクセントは teal `#14B87C` の1色に集約**（ナビ/ボタン/チャート/リアクション/達成リング）。色を増やさない。
- カラートークン（現行を正とする。古い spec 値には寄せない）：
  - bg `#FAFAF8` / card `#FFFFFF` / line（ヘアライン）`#ECEBE7`
  - ink（文字）`#181D1A` / sub `#6E736C` / faint `#A2A69F`
  - accent `#14B87C` / asoft `#E8F6EF` / aline `#CDEBDD`
- 書体：Plus Jakarta Sans + Noto Sans JP。
- **アバターは頭文字のみ・単色円**（グラデ不可・絵文字フェイス不可）。
- データ表示は「HUD風の小さなデータ行」で情報量を上げつつ、余白で疲れさせない。

## 部位タグ色
| 部位 | 色 |
|---|---|
| 胸トレ | `#FF6A3D` |
| 背中 | `#3E86C9` |
| 脚 | `#7C6CD0` |
| 肩・腕 | `#E0A53A` |
| 有酸素 | `#14B87C`（アクセントと統一） |
| 休養 | `#9AA09A` |

タグのステータス表現：`done`＝asoft背景＋teal✓ / `changed`＝アンバー「変更」バッジ / `todo`＝破線・薄色。

## やらないこと
- **黒背景・ネオン・近未来テーマは不採用**（現行のミニマル方向と喧嘩するため）。
- カロリー収支グラフは「のび」に置かない（収支は記録/Feed側のHUDで表現）。
- 連続記録は個人ベース（グループ集計しない）。
- **バックエンド・認証・localStorage/PWA化は次フェーズ**（現状は静的プロトタイプ）。

## 技術構成 / 公開先
- **単一の静的 `index.html`**。Tailwind と Chart.js は CDN 読み込み。ビルド工程なし。
- 公開先：**GitHub Pages**（`.github/workflows/deploy.yml` で `main` push 時に自動デプロイ）。
- 単一HTMLのため内部アセットの相対パス参照はなく、サブパス公開でもリンクは壊れない。
- 将来 Vite 等を導入する場合は、deploy ワークフローの artifact path を `dist` に変え、ビルドステップを追加する。
